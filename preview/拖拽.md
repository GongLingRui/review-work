# 2D编辑器中两种拖拽方式的实现原理

## 1. 外部文件拖拽：从系统拖拽文件到上传区域

外部文件拖拽是通过HTML5的拖放API实现的，主要涉及以下几个关键步骤：

### 1.1 拖拽区域定义

在`MainUiLeftUpload.tsx`中，通过为容器元素添加拖拽相关事件监听器来实现拖拽区域：

```typescript
<div
  className={classes.listWrap}
  style={{ paddingRight: allRight_state?.state ? 20 : 0 }}
  onDragOver={handleDragOver}
  onDrop={handleDrop}
>
  {/* 内容省略 */}
</div>
```

### 1.2 拖拽悬停处理

`handleDragOver`函数用于处理文件拖到区域上方时的行为：

```typescript
const handleDragOver = useCallback((event: React.DragEvent<HTMLDivElement>) => {
  event.preventDefault(); // 阻止默认行为
}, []);
```

这里的`event.preventDefault()`非常关键，它告诉浏览器该区域可以接受拖放，如果不阻止默认行为，浏览器会禁止放置操作。

### 1.3 拖拽放置处理

`handleDrop`函数处理文件放置时的逻辑：

```typescript
const handleDrop = useCallback((action: React.DragEvent<HTMLDivElement>) => {
  action.stopPropagation(); // 阻止冒泡
  action.preventDefault(); // 阻止默认行为
  const files = action.dataTransfer.files; // 获取拖拽的文件
  
  if (files.length > 0) {
    Array.from(files as FileList).forEach((item) => {
      if (!checkFileSize(item)) return; // 检查文件大小
      
      // 处理文件
      const fileExtension = item.name.split('.').pop()?.toLowerCase();
      const newName = 'uploadImage.' + fileExtension;
      const newFile = new File([item], newName, { type: item.type });
      
      // 上传文件
      uploadImageForCavas({
        // 配置和回调函数
        updateStart: () => editorToastShow({...}),
        updateEnd: (ret: boolean, error: number, message?: string) => {...},
        fileExtension: fileExtension as string,
        fileItem: newFile,
        event: event,
        canvasEditor: canvasEditor,
        uploadFileType: GetUpTokenFileTypeEnum.Edit2dLocal,
      });
    });
  } else {
    setNetLoading(false);
  }
}, [canvasEditor]);
```

这个函数的关键步骤包括：
1. 阻止事件冒泡和默认行为
2. 通过`dataTransfer.files`获取拖拽的文件列表
3. 遍历文件列表，对每个文件进行处理：
   - 检查文件大小
   - 获取文件扩展名
   - 创建新的文件对象（重命名）
   - 调用上传函数处理文件

## 2. 内部素材拖拽：从素材列表拖拽到画布

内部素材拖拽同样使用HTML5拖放API，但实现方式更简单，主要在`DataList.tsx`中：

### 2.1 设置素材项为可拖拽

```typescript
<div
  key={v.material_id}
  id={decodedUrl}
  className={classes.item}
  onMouseEnter={() => handleMouseEnter(v.material_id)}
  onMouseLeave={() => handleMouseLeave(v.material_id)}
  draggable={true}  // 关键属性，使元素可拖拽
  onDragEnd={() => clickItem(v)}  // 拖拽结束时的处理
  onDragStart={() => {  // 拖拽开始时的处理
    setHoverStates((prev) => ({ ...prev, [v.material_id]: false }))
  }}
>
  <CommonImage
    src={decodedUrl}
    style={{...}}
    onClick={() => clickImgItem(v)}
  />
  {/* 其他UI元素 */}
</div>
```

### 2.2 拖拽开始处理

当用户开始拖拽素材时，`onDragStart`事件触发：

```typescript
onDragStart={() => {
  setHoverStates((prev) => ({ ...prev, [v.material_id]: false }))
}}
```

这里主要是清除当前素材的悬停状态，隐藏一些UI元素（如更多操作按钮），使拖拽过程更加干净。

### 2.3 拖拽结束处理

当用户完成拖拽（释放鼠标）时，`onDragEnd`事件触发：

```typescript
onDragEnd={() => clickItem(v)}
```

这里直接调用`clickItem`函数，与点击素材时的处理逻辑相同。这个函数负责将素材添加到画布上：

```typescript

  } else if (isPetPoratrait) {
    handlePetPoraData(data);
  } else {
    /// 导入到画布
    let decodedUrl = /* 获取URL */
    const fileExtension = data.file_name.split('.').pop(); // 获取文件后缀
    if (fileExtension === 'svg') {
      // SVG 文件的处理逻辑
      const response = await fetch(decodedUrl);
      const blob = await response.blob();

        canvasEditor?.addSvgFile(file as string);
      });
    } else {

      const base64 = await convertToBase64(decodedUrl);

          importSource: ImportSource.Cloud,
          fileType: fileExtension,
          key_prefix: data.file_name
        });
      setNetLoading(false);
    }
  }
}
```

## 两种拖拽方式的区别

1. **实现复杂度**：
   - 外部文件拖拽：需要处理文件对象、检查文件类型和大小、上传文件等
   - 内部素材拖拽：实现更简单，只需设置元素为可拖拽并处理拖拽结束事件

2. **数据处理**：
   - 外部文件拖拽：处理的是原始文件对象，需要转换为适合画布的格式
   - 内部素材拖拽：处理的是已存在于系统中的素材数据

3. **拖拽目标**：
   - 外部文件拖拽：目标是上传区域，文件会先上传再添加到画布
   - 内部素材拖拽：目标是画布，素材会直接添加到画布

4. **事件处理**：
   - 外部文件拖拽：需要处理`onDragOver`和`onDrop`事件
   - 内部素材拖拽：主要处理`onDragStart`和`onDragEnd`事件

这两种拖拽方式共同为用户提供了灵活的素材添加方式，使2D编辑器的操作更加直观和高效。
