# 上传代码逻辑详解

这段代码实现了一个文件上传的功能模块，主要通过XMLHttpRequest实现文件上传到服务器的过程。整个上传流程分为两个主要步骤：获取上传凭证和执行上传操作。下面详细分析其逻辑：

## 1. 核心函数

代码中定义了三个核心函数：
- `getBody`: 解析XMLHttpRequest响应
- `getUploadToken`: 获取上传凭证
- `xhrUpload`: 执行实际的文件上传
- `uploadRequest`: 整合上述流程的主函数

## 2. 获取响应体 - getBody

```typescript
function getBody(xhr: XMLHttpRequest) {
  const text = xhr.responseText || xhr.response;

  if (!text) {
    return text;
  }

  try {
    return JSON.parse(text);
  } catch (e) {
    return text;
  }
}
```

这个函数负责从XMLHttpRequest对象中提取响应内容：
- 获取响应文本
- 尝试将文本解析为JSON对象
- 如果解析失败，则返回原始文本

## 3. 获取上传凭证 - getUploadToken

```typescript
const getUploadToken = async (options: RequestOptions) => {
  const {
    onError = NOOP,
    file,
  } = options;
  let resp
  if (!options.uploadFileType) {
    onError()
    return
  }

  try {
    resp = await getUpToken({ file_name: file.name, file_type: options.uploadFileType })

    if (!resp?.data?.up_token || !resp?.data?.key_prefix) {
      onError()
      return
    }
    return {
      up_token: resp?.data?.up_token,
      key_prefix: resp?.data?.key_prefix,
    }
  } catch (error) {
    onError(error)
    ConsoleUtil.error(error)
    return
  }
}
```

这个函数负责从服务器获取上传凭证：
- 从options中提取文件和错误回调
- 检查是否提供了uploadFileType，如果没有则调用错误回调并返回
- 调用getUpToken API获取上传凭证，传入文件名和文件类型
- 验证响应中是否包含必要的up_token和key_prefix
- 如果验证通过，返回包含up_token和key_prefix的对象
- 如果过程中出现错误，调用错误回调并记录错误

## 4. 执行上传 - xhrUpload

```typescript
const xhrUpload = (options: RequestOptions, uploadRes: { up_token: string; key_prefix: string }) => {
  const {
    onProgress = NOOP,
    onError = NOOP,
    onSuccess = NOOP,
    file,
  } = options;
  if (typeof XMLHttpRequest !== "undefined") { // 修复 gatsby ssr 构建报错问题
    const xhr = new XMLHttpRequest();
    if (onProgress && xhr.upload) {
      xhr.upload.onprogress = function (event: ProgressEvent) {
        let percent;
        if (event.total > 0) {
          percent = (event.loaded / event.total) * 100;
        }
        const _percent = parseInt(percent, 10)
        if (_percent % 5 === 0) {
          onProgress(_percent, event);
        }
      };
    }
    xhr.onerror = function error(e) {
      onError(e);
    };
    xhr.onload = function onload() {
      if (xhr.status < 200 || xhr.status >= 300) {
        return onError(getBody(xhr));
      }

      onSuccess({ body: getBody(xhr), key_prefix: uploadRes.key_prefix });
    };

    // 开始上传
    xhr.open('PUT', uploadRes.up_token, true);
    xhr.send(file);
    return {
      abort() {
        xhr.abort();
      },
    };
  }
}
```

这个函数负责实际的文件上传操作：
- 从options中提取进度回调、错误回调、成功回调和文件
- 检查XMLHttpRequest是否可用（防止服务端渲染环境报错）
- 创建XMLHttpRequest实例
- 设置上传进度监听器，计算上传百分比并每增加5%调用一次进度回调
- 设置错误处理函数
- 设置加载完成处理函数，检查HTTP状态码并调用相应的回调
- 使用PUT方法打开请求，URL为上传凭证中的up_token
- 发送文件
- 返回一个包含abort方法的对象，用于取消上传

## 5. 整合流程 - uploadRequest

```typescript
const uploadRequest: UploadRequest = async (options: RequestOptions) => {
  try {
    const resp = await getUploadToken(options)
    if (!resp) {
      throw new Error('get upload token error')
      return
    }
    const request = xhrUpload(options, resp)
    return request
  } catch (error) {
    return {
      abort() {

      },
    };
  }
};
```

这个函数整合了整个上传流程：
- 调用getUploadToken获取上传凭证
- 如果获取凭证失败，抛出错误
- 调用xhrUpload执行实际的上传操作
- 返回上传请求对象，包含abort方法用于取消上传
- 如果过程中出现错误，返回一个空的abort方法

## 6. 上传流程总结

1. 客户端调用uploadRequest函数，传入文件和回调函数
2. uploadRequest调用getUploadToken获取上传凭证
3. 获取凭证成功后，调用xhrUpload执行实际的上传操作
4. 上传过程中，根据进度调用进度回调
5. 上传完成后，调用成功回调或错误回调
6. 整个过程中，客户端可以通过返回的abort方法取消上传

这种设计将上传过程分为获取凭证和执行上传两个独立步骤，使代码结构清晰，同时提供了进度监控和错误处理机制，适合处理大文件上传场景。
