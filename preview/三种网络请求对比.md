# XMLHttpRequest、Fetch 和 Axios 的区别及选择原因

在前端开发中，XMLHttpRequest、Fetch 和 Axios 是三种常用的网络请求方式，它们各有特点和适用场景。

## 三者的主要区别

### 1. XMLHttpRequest (XHR)

XMLHttpRequest 是最早的 AJAX 请求方法，具有以下特点：
- 是浏览器原生 API，兼容性最好
- 使用回调函数处理响应，容易导致回调地狱
- 配置相对复杂，需要手动设置各种事件监听器
- 提供了丰富的事件（如 onprogress），适合处理文件上传和下载进度监控
- 支持请求取消

### 2. Fetch API

Fetch 是较新的浏览器原生 API，基于 Promise：
- 语法更简洁，基于 Promise，避免回调地狱
- 原生支持，不需要额外依赖
- 默认不发送 Cookie
- 不支持请求超时设置
- 不能原生监听上传进度
- 早期版本不支持请求取消（现在通过 AbortController 支持）

### 3. Axios

Axios 是基于 Promise 的 HTTP 客户端库：
- 封装了 XMLHttpRequest，提供了更友好的 API
- 支持浏览器和 Node.js 环境
- 自动转换 JSON 数据
- 支持请求和响应拦截
- 支持请求取消
- 支持请求超时
- 支持上传进度监控
- 需要额外引入依赖

## 为什么上传代码使用 XMLHttpRequest 而不是其他方式

在上传文件的场景中，代码选择使用 XMLHttpRequest 而非 Fetch 或 Axios 主要有以下原因：

1. **进度监控**：XMLHttpRequest 提供了原生的 `onprogress` 事件，可以精确监控上传进度。代码中通过这个事件计算上传百分比并每增加 5% 触发一次回调：

```typescript
xhr.upload.onprogress = function (event: ProgressEvent) {
  let percent;
  if (event.total > 0) {
    percent = (event.loaded / event.total) * 100;
  }
  const _percent = parseInt(percent, 10)
  if (_percent % 5 === 0) {
    onProgress(_percent, event);
  }
};
```

2. **灵活控制**：XMLHttpRequest 提供了更底层的控制能力，可以更灵活地处理各种上传场景。

3. **请求取消**：代码中返回了一个包含 `abort()` 方法的对象，允许调用者随时取消上传：

```typescript
return {
  abort() {
    xhr.abort();
  },
};
```

4. **兼容性考虑**：XMLHttpRequest 的兼容性比 Fetch 更好，特别是在处理文件上传时。

5. **直接使用**：在上传文件这种特定场景下，直接使用 XMLHttpRequest 可能比引入 Axios 这样的库更轻量，避免了不必要的依赖。

而在项目的其他部分（如 `requestOld.ts` 文件中），使用了 Fetch API 来处理一般的 HTTP 请求，这是因为这些请求不需要监控进度，且 Fetch 的 Promise 接口更易于使用和维护。

总的来说，选择哪种请求方式应该根据具体需求来决定：
- 需要监控上传进度或有特殊控制需求时，选择 XMLHttpRequest
- 一般的数据请求，可以使用更现代的 Fetch API
- 需要更多功能且不介意引入依赖时，可以选择 Axios

在这个项目中，上传功能选择 XMLHttpRequest 是合理的，因为它需要精确的进度监控和取消能力。
