# 文章处理流程测试文档

## 已修复的问题

### 1. 后端修复

#### 1.1 数据加载问题
- **问题**：`get_stories_by_status` 返回 QuerySet 切片，需要转换为列表
- **修复**：在 `Story.get_stories_by_status` 方法中添加 `list()` 转换

#### 1.2 文章内容获取问题
- **问题**：从 `get_stories_by_status` 获取的 Story 对象可能没有完整内容（可能被 defer）
- **修复**：在处理前重新从数据库加载完整的 Story 对象

#### 1.3 新文章标记问题
- **问题**：`bulk_create` 后的 `CommonStory` 对象没有 `process_status` 属性
- **修复**：通过 `unique_id` 查找对应的 `Story` 对象并更新状态

#### 1.4 AI API 重试机制
- **问题**：`initial_review_story` 和 `analyze_segment` 方法缺少重试机制
- **修复**：添加了完整的重试机制（3次重试，指数退避）

### 2. 前端修复

#### 2.1 处理状态组件
- **问题**：连接线定位不正确
- **修复**：调整了连接线的定位逻辑，使用相对定位

#### 2.2 API 客户端
- **问题**：缺少获取处理状态的 API 方法
- **修复**：添加了 `storyApi.getProcessStatus()` 方法

## 常见问题排查

### 问题1：文章没有被标记为待初评
- **原因**：可能是新文章没有内容，或者 `process_status` 字段已有值
- **解决**：检查 `harbor_service.py` 中的标记逻辑，确保新文章有内容

### 问题2：AI API 调用失败
- **原因**：API Key 未配置，或网络问题
- **解决**：检查 `.env` 文件中的 `RSSANT_AI_API_KEY` 配置

### 问题3：处理流程卡住
- **原因**：可能是某个步骤的异常处理不当
- **解决**：检查日志，查看具体错误信息

## 配置要求

### 环境变量
```bash
RSSANT_AI_API_KEY=your_api_key
RSSANT_AI_API_BASE_URL=https://open.bigmodel.cn/api/paas/v4
RSSANT_AI_TRANSLATE_MODEL_CONFIG=glm-z1-flash,glm-z1-flash
```

### 定时任务
确保 scheduler 服务正常运行，定时任务会自动执行：
- 每5分钟：处理待初评文章
- 每10分钟：处理待分析文章
- 每15分钟：处理待翻译文章

## 性能优化建议

1. **批量处理**：可以增加每次处理的数量限制
2. **并发控制**：避免同时处理过多文章导致API限流
3. **缓存机制**：对于已分析的文章，可以缓存分析结果

## 监控指标

建议监控以下指标：
- 待初评文章数量
- 待分析文章数量
- 待翻译文章数量
- AI API 调用成功率
- 处理耗时
