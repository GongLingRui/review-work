## 📋 服务管理

### 查看服务状态
```bash
bash check-services.sh
```

### 查看服务日志
```bash
# API 服务日志
tail -f /tmp/rssant-api.log

# Worker 服务日志
tail -f /tmp/rssant-worker.log

# Scheduler 服务日志
tail -f /tmp/rssant-scheduler.log

# 前端服务日志
tail -f /tmp/rssant-frontend.log
```

### 停止服务
```bash
# 停止所有服务
pkill -f "runserver"
pkill -f "run-scheduler"
pkill -f "vite"

# 停止数据库
docker stop rssant-postgres
```

### 重启服务
```bash
# 重启 API 服务
pkill -f "rundev-api"
cd /Users/gongfan/Desktop/公司项目/rss-subscribe
source $(conda info --base)/etc/profile.d/conda.sh
conda activate rssant
nohup bash scripts/dev/rundev-api.sh > /tmp/rssant-api.log 2>&1 &

# 重启 Scheduler 服务
pkill -f "run-scheduler"
nohup bash scripts/dev/rundev-scheduler.sh > /tmp/rssant-scheduler.log 2>&1 &
```

## 📝 注意事项

1. **数据库**: PostgreSQL 通过 Docker 运行，确保 Docker Desktop 正在运行
2. **环境**: 所有后端服务需要在 `rssant` conda 环境中运行
3. **日志**: 所有服务日志保存在 `/tmp/rssant-*.log`
4. **端口**: 确保端口 6788、6793、5173、5432 未被其他程序占用

## 🐛 故障排查

### 如果服务无法启动

1. **检查数据库是否运行**
   ```bash
   docker ps | grep postgres
   lsof -i :5432
   ```

2. **检查 conda 环境**
   ```bash
   conda activate rssant
   python --version
   ```

3. **检查依赖是否安装**
   ```bash
   conda activate rssant
   python -c "import django; print(django.get_version())"
   ```

4. **查看错误日志**
   ```bash
   tail -50 /tmp/rssant-api.log
   tail -50 /tmp/rssant-scheduler.log
   ```
