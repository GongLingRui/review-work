# 飞书机器人发送功能测试指南

## 测试方法

### 方法1：使用测试脚本（推荐）

1. **激活conda虚拟环境**：
   ```bash
   conda activate 你的环境名
   # 或者
   source activate 你的环境名
   ```

2. **运行测试脚本**：
   ```bash
   python test_feishu_bot_send.py
   ```

3. **或者使用Django管理命令**：
   ```bash
   python manage.py shell < test_feishu_bot_send.py
   ```

### 方法2：通过前端界面测试

1. 访问飞书机器人配置页面
2. 找到已启用的配置
3. 点击"测试"按钮
4. 检查飞书群是否收到消息

### 方法3：使用API测试

#### 测试单个配置

使用 `feishu_bot.test` API：
```python
POST /api/v1/feishu_bot/test
{
    "config_id": 1,  # 配置ID，可选
    "webhook_url": "https://open.feishu.cn/open-apis/bot/v2/hook/xxx",  # 可选
    "report_type": "ai_entertainment"  # 或 "aigc"，可选
}
```

#### 测试定时任务发送

使用 `harbor_rss.feishu_bot_send_ai_reports` API：
```python
POST /api/v1/harbor_rss.feishu_bot_send_ai_reports
```

**注意**：这个API会检查当前时间是否匹配配置的发送时间，如果不在发送时间范围内，不会发送。

## 测试检查项

### 1. 配置检查
- ✅ 是否有启用的飞书机器人配置
- ✅ 配置的webhook_url是否有效
- ✅ 配置的发送时间格式是否正确（HH:MM，多个时间用逗号分隔）

### 2. 报告内容检查
- ✅ 是否有昨天的AI影视报告内容
- ✅ 是否有昨天的AIGC报告内容
- ✅ 报告内容是否完整（有摘要和参考链接）

### 3. 发送功能检查
- ✅ 能否成功发送消息到飞书群
- ✅ 消息格式是否正确（卡片消息）
- ✅ 消息内容是否包含摘要和参考链接

### 4. 定时任务检查
- ✅ 定时任务是否每分钟执行
- ✅ 时间匹配逻辑是否正确
- ✅ 重复发送检查是否正确

## 常见问题

### Q1: 测试时提示"当前时间不匹配配置的发送时间"

**A**: 这是正常的。定时任务会检查当前时间是否在配置的发送时间范围内（允许前后1分钟容差）。如果想强制测试，可以使用前端的"测试"按钮或API的`feishu_bot.test`接口。

### Q2: 测试时提示"报告不存在"

**A**: 需要先生成报告内容。可以：
1. 等待定时任务在凌晨1点自动生成
2. 或者手动调用 `harbor_rss.feishu_generate_daily_content_force` API强制生成

### Q3: 测试发送成功但飞书群没收到消息

**A**: 检查：
1. Webhook URL是否正确
2. 飞书机器人是否被移除或失效
3. 查看日志中的错误信息

### Q4: 多个发送时间点（如09:00,17:00）只有第一个发送了

**A**: 这个问题已经修复。现在每个时间点都会正确发送。如果仍有问题，请检查：
1. 日志中的时间匹配信息
2. `last_sent_at`字段是否正确更新
3. 是否有重复发送的检查错误

## 修复的问题

### 1. 修复了daily频率多个发送时间的重复发送检查逻辑
- **问题**：当配置了多个发送时间（如09:00和17:00）时，在09:00发送后，由于检查逻辑只检查报告日期，导致17:00不会发送。
- **修复**：改为检查是否在今天已经在这个特定时间点发送过，而不是简单地检查报告日期。

### 2. 修复了last_sent_report_date的错误日期比较逻辑
- **问题**：存在错误的日期比较检查。
- **修复**：移除了错误的检查，改进了重复发送检查逻辑。

### 3. 优化了时间匹配逻辑
- **修复**：保留精确匹配和容差匹配逻辑（前后1分钟容差），改进了重复发送检查。

### 4. 添加了详细的日志记录
- **修复**：添加了配置详情、发送过程和时间匹配的详细日志。

## 日志查看

```bash
tail -f logs/scheduler.log | grep "飞书机器人"
```

或者查看详细的发送日志：
```bash
tail -f logs/scheduler.log | grep -E "飞书|feishu|Feishu" -i
```
