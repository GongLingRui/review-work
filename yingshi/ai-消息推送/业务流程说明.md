# 业务流程说明

## 全量设为已读流程

```mermaid
flowchart TD
    A[触发: feed.set_all_readed] --> B[获取用户所有订阅]
    B --> C{存在未读文章?}
    C -- 否 --> D[结束]
    C -- 是 --> E[更新UserFeed.story_offset到total_storys]
    E --> F[查询未读offset范围]
    F --> G[批量查找已存在的UserStory]
    G --> H{存在未标记记录?}
    H -- 是 --> I[批量更新: is_watched=true, dt_watched=now]
    H -- 否 --> J[准备需要创建的UserStory列表]
    I --> K[事务提交]
    J --> K[事务提交]
    K --> L[返回更新结果]
```

- 实现位置：`rssant_api/models/union_feed.py:342`。

## 文章导出到飞书文档流程

```mermaid
flowchart TD
    A[触发: story.export_to_feishu] --> B[检查配置: app_id/app_secret]
    B -->|缺失| C[返回错误]
    B -->|齐备| D[创建FeishuService实例]
    D --> E[权限校验: feed_unionids]
    E --> F[遍历文章: 获取详情]
    F --> G{文章存在?}
    G -->|否| H[记录警告并跳过]
    G -->|是| I[调用导出接口创建文档]
    I --> J[收集结果: 文档ID/URL/标题]
    J --> K[汇总并返回]
```

- 实现位置：`rssant_api/views/story.py:336`。

## 订阅抓取与更新流程（简化）

```mermaid
flowchart TD
    A[定时触发] --> B[查询过期订阅: take_outdated_feeds]
    B --> C[批量更新为pending并返回抓取参数]
    C --> D[worker抓取RSS]
    D --> E{成功?}
    E -- 否 --> F[状态error, 记录响应]
    E -- 是 --> G[更新Feed与Story集合]
    G --> H[刷新统计与时间字段]
    H --> I[结束]
```

- 实现位置：`rssant_api/models/feed.py:246`、`rssant_api/models/story.py:253`。
