# RSS订阅服务启动说明

## 问题诊断

如果导入订阅后，任务一直显示"处理中"（pending），通常是因为 **Scheduler 服务没有运行**。

## 服务架构

RSS订阅系统需要运行以下服务：

1. **API 服务** (`rundev-api.sh`) - 处理前端请求，端口 6788
2. **Worker 服务** (`rundev-worker.sh`) - 执行实际的任务（如抓取订阅源），端口 6793
3. **Scheduler 服务** (`rundev-scheduler.sh`) - **关键服务**，从任务队列获取任务并分发给 Worker，端口 6790
4. **AsyncAPI 服务** (`run-asyncapi.py`) - 异步API服务

## 启动所有服务

在 conda 虚拟环境中，分别启动以下服务：

### 1. 启动 API 服务
```bash
cd /root/gongfan/rss-subscribe
source $(conda info --base)/etc/profile.d/conda.sh
conda activate rssant
bash rundev-api.sh
```

### 2. 启动 Worker 服务
```bash
# 新开一个终端
cd /root/gongfan/rss-subscribe
source $(conda info --base)/etc/profile.d/conda.sh
conda activate rssant
bash rundev-worker.sh
```

### 3. 启动 Scheduler 服务（重要！）
```bash
# 新开一个终端
cd /root/gongfan/rss-subscribe
source $(conda info --base)/etc/profile.d/conda.sh
conda activate rssant
bash rundev-scheduler.sh
```

### 4. 启动 AsyncAPI 服务（可选）
```bash
# 新开一个终端
cd /root/gongfan/rss-subscribe
source $(conda info --base)/etc/profile.d/conda.sh
conda activate rssant
python run-asyncapi.py
```

## 使用 screen 或 tmux 管理多个服务

### 使用 screen
```bash
# 安装 screen（如果未安装）
sudo apt-get install screen

# 创建 screen 会话
screen -S rssant-api
# 在 screen 中启动 API 服务
bash rundev-api.sh
# 按 Ctrl+A 然后 D 退出 screen（服务继续运行）

screen -S rssant-worker
bash rundev-worker.sh
# 按 Ctrl+A 然后 D 退出

screen -S rssant-scheduler
bash rundev-scheduler.sh
# 按 Ctrl+A 然后 D 退出

# 查看所有 screen 会话
screen -ls

# 重新连接到会话
screen -r rssant-api
```

### 使用 tmux
```bash
# 安装 tmux（如果未安装）
sudo apt-get install tmux

# 创建 tmux 会话
tmux new -s rssant-api
# 在 tmux 中启动 API 服务
bash rundev-api.sh
# 按 Ctrl+B 然后 D 退出 tmux

tmux new -s rssant-worker
bash rundev-worker.sh
# 按 Ctrl+B 然后 D 退出

tmux new -s rssant-scheduler
bash rundev-scheduler.sh
# 按 Ctrl+B 然后 D 退出

# 查看所有 tmux 会话
tmux ls

# 重新连接到会话
tmux attach -t rssant-api
```

## 检查服务状态

```bash
# 检查进程
ps aux | grep -E "runserver|run-scheduler|run-asyncapi" | grep -v grep

# 检查端口
netstat -tuln | grep -E "6788|6790|6793"
```

## 任务处理流程

1. 用户导入订阅 → API 服务创建 `FeedCreation` 记录（状态：pending）
2. API 服务调用 `batch_find_feed` → 创建 `WorkerTask` 任务到数据库队列
3. **Scheduler 服务**从数据库队列获取任务
4. Scheduler 调用 Worker 服务执行任务
5. Worker 服务处理完成后，更新 `FeedCreation` 状态为 `ready` 或 `error`
6. 前端自动刷新显示已完成的订阅

## 常见问题

### Q: 任务一直处于 pending 状态？
A: 检查 Scheduler 服务是否运行：
```bash
ps aux | grep run-scheduler
```
如果没有运行，启动它：
```bash
bash rundev-scheduler.sh
```

### Q: 如何查看任务队列？
A: 使用 Django shell：
```bash
python manage.py shell
>>> from rssant_api.models import WorkerTask
>>> WorkerTask.objects.count()  # 查看任务总数
>>> WorkerTask.stats()  # 查看任务统计
```

### Q: 如何查看日志？
A: 查看各个服务的控制台输出，或者查看日志文件（如果配置了日志文件）。

## 配置说明

服务配置在环境变量或配置文件中：
- `RSSANT_ROLE`: 服务角色（api/worker/scheduler/asyncapi）
- `RSSANT_BIND_ADDRESS`: 绑定地址
- `RSSANT_HARBOR_URL`: Harbor 服务地址（默认 http://localhost:6788）
- `RSSANT_WORKER_URL`: Worker 服务地址（默认 http://localhost:6793）
