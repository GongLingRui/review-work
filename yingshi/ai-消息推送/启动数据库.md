# 启动PostgreSQL数据库

## 问题

登录时出现500错误，通常是因为PostgreSQL数据库未运行。

## 解决方案

### 方法1: 使用Docker启动PostgreSQL（推荐）

1. **启动Docker Desktop**
   - 在macOS上，打开Docker Desktop应用
   - 等待Docker完全启动（状态栏显示Docker图标）

2. **启动PostgreSQL容器**
   ```bash
   cd /path/to/rss-subscribe
   bash scripts/postgres_start.sh
   ```

3. **验证数据库是否运行**
   ```bash
   # 检查端口是否监听
   lsof -iTCP -sTCP:LISTEN -n -P | grep 5432
   
   # 或检查Docker容器
   docker ps | grep postgres
   ```

### 方法2: 使用Homebrew安装并启动PostgreSQL

1. **安装PostgreSQL**
   ```bash
   brew install postgresql@11
   ```

2. **启动PostgreSQL服务**
   ```bash
   brew services start postgresql@11
   ```

3. **创建数据库和用户**
   ```bash
   # 进入PostgreSQL命令行
   psql postgres
   
   # 在PostgreSQL中执行
   CREATE USER rssant WITH PASSWORD 'rssant';
   CREATE DATABASE rssant OWNER rssant;
   \q
   ```

4. **运行数据库迁移**
   ```bash
   conda activate rssant
   python manage.py migrate
   ```

### 方法3: 使用系统PostgreSQL

如果系统已安装PostgreSQL：

1. **启动PostgreSQL服务**
   ```bash
   # macOS (使用launchctl)
   sudo launchctl load -w /Library/LaunchDaemons/com.edb.launchd.postgresql-*.plist
   
   # 或使用pg_ctl
   pg_ctl -D /usr/local/var/postgres start
   ```

2. **创建数据库和用户**（同方法2）

## 验证数据库连接

```bash
# 测试数据库连接
conda activate rssant
python manage.py dbshell

# 如果连接成功，会进入PostgreSQL命令行
# 输入 \q 退出
```

## 检查数据库配置

数据库配置在 `rssant_config/env.py` 中，默认值：
- Host: localhost
- Port: 5432
- Database: rssant
- User: rssant
- Password: rssant

可以通过环境变量修改：
```bash
export RSSANT_PG_HOST=localhost
export RSSANT_PG_PORT=5432
export RSSANT_PG_DB=rssant
export RSSANT_PG_USER=rssant
export RSSANT_PG_PASSWORD=rssant
```

## 常见问题

### Q: Docker无法连接
A: 确保Docker Desktop正在运行，检查：
```bash
docker ps
```

### Q: 端口5432已被占用
A: 检查并停止占用端口的进程：
```bash
lsof -i :5432
# 然后停止相关进程
```

### Q: 数据库连接被拒绝
A: 检查PostgreSQL服务是否运行：
```bash
# Docker方式
docker ps | grep postgres

# Homebrew方式
brew services list | grep postgres
```

## 启动所有服务

数据库启动后，确保所有服务正常运行：

```bash
# 启动所有后端服务
bash start-all-services.sh start

# 检查服务状态
bash start-all-services.sh status
```
