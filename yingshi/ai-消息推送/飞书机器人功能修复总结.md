# 飞书机器人功能修复总结

## 修复时间
2025-11-20

## 修复的问题

### 1. ✅ 报告日期不匹配问题
**问题**：发送函数使用的是 `today` 的报告，但定时任务生成的是 `yesterday` 的报告。

**修复**：
- 将发送函数改为使用 `yesterday` 获取报告
- 确保发送的是昨天已生成的报告，而不是当天的报告

### 2. ✅ 报告模型类型错误
**问题**：发送AI影视报告时使用了错误的模型 `AIEntertainmentReport`，应该使用 `AIEntertainmentOnlyReport`。

**修复**：
- AI影视报告：使用 `AIEntertainmentOnlyReport.get_by_date(yesterday)`
- AIGC报告：使用 `AIGCReport.get_by_date(yesterday)`

### 3. ✅ 重复调用搜索API问题
**问题**：发送消息时如果报告不存在，会再次调用搜索API生成报告，导致重复调用API。

**修复**：
- 移除了发送时的报告生成逻辑
- 如果报告不存在，直接跳过发送并记录警告日志
- 确保只使用已生成的报告，不重复调用搜索API

### 4. ✅ 缺少定时任务
**问题**：没有定时任务来调用发送函数。

**修复**：
- 在 `rssant_scheduler/timer_task.py` 中添加了定时任务
- 每分钟检查一次，根据用户配置的发送时间发送昨天的报告

### 5. ✅ 报告日期显示问题
**修复**：
- 在飞书卡片中显示报告日期（昨天的日期）
- 更新 `last_sent_report_date` 为 `yesterday` 而不是 `today`

## 修复后的工作流程

### 报告生成流程
1. **每天凌晨1点**：定时任务 `harbor_rss.ai_entertainment_generate_yesterday_report` 和 `harbor_rss.aigc_generate_yesterday_report` 执行
2. **生成昨天的报告**：调用搜索API（Tavily或百度AppBuilder）生成报告
3. **保存报告**：将报告保存到数据库（`AIEntertainmentOnlyReport` 或 `AIGCReport`）

### 消息发送流程
1. **每分钟检查**：定时任务 `harbor_rss.feishu_bot_send_ai_reports` 每分钟执行一次
2. **检查配置**：遍历所有启用的飞书机器人配置
3. **判断是否发送**：
   - 检查当前时间是否匹配配置的发送时间
   - 检查发送频率（每天或每周）
   - 检查是否已经发送过（避免重复发送）
4. **获取报告**：从数据库获取昨天的报告（`AIEntertainmentOnlyReport` 或 `AIGCReport`）
5. **发送消息**：如果报告存在，构建飞书卡片消息并发送到配置的Webhook URL
6. **更新状态**：记录发送时间和报告日期

## 关键代码修改

### 1. `rssant_harbor/view.py` - `do_feishu_bot_send_ai_reports`

**修改前**：
```python
today = now.date()
report = AIEntertainmentReport.get_by_date(today)
if not report:
    # 生成报告（会调用搜索API）
    ...
```

**修改后**：
```python
yesterday = (now - timedelta(days=1)).date()
report = AIEntertainmentOnlyReport.get_by_date(yesterday)
if not report:
    LOG.warning(f'AI影视报告不存在（日期: {yesterday}），跳过发送: {cfg.name}')
    continue  # 不生成报告，直接跳过
```

### 2. `rssant_scheduler/timer_task.py`

**添加**：
```python
# 飞书机器人消息发送定时任务（每分钟检查一次，根据用户配置的发送时间发送）
dict(
    api='harbor_rss.feishu_bot_send_ai_reports',
    timer=Timer('1m'),  # 每分钟检查一次
),
```

## 注意事项

1. **搜索API调用限制**：
   - 报告只在凌晨1点生成一次（调用搜索API）
   - 发送消息时不再调用搜索API，只使用已生成的报告
   - 这样可以大大减少API调用次数

2. **报告日期**：
   - 生成的是昨天的报告
   - 发送的也是昨天的报告
   - 确保用户收到的是完整的一天数据

3. **发送时间**：
   - 系统每分钟检查一次
   - 如果当前时间匹配配置的发送时间，就会发送
   - 支持多个时间点（用逗号分隔）

4. **重复发送保护**：
   - 检查 `last_sent_at` 避免在同一分钟重复发送
   - 记录 `last_sent_report_date` 跟踪发送的报告日期
