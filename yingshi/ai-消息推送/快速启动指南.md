# RSS订阅项目 - 快速启动指南

## 🚀 一键启动

```bash
cd /Users/gongfan/Desktop/公司项目/rss-subscribe
bash start-project.sh
```

## 📋 启动前检查清单

- [ ] Docker Desktop 正在运行
- [ ] Conda 环境 `rssant` 已创建
- [ ] Python 依赖已安装 (`pip install -r requirements.txt`)
- [ ] 前端依赖已安装 (`cd frontend && npm install`)
- [ ] `.env` 文件已配置

## 🔧 手动启动步骤

### 1. 启动数据库
```bash
bash scripts/postgres_start.sh
```

### 2. 激活环境并启动后端服务
```bash
source $(conda info --base)/etc/profile.d/conda.sh
conda activate rssant

# 启动API服务
nohup bash scripts/dev/rundev-api.sh > /tmp/rssant-api.log 2>&1 &

# 启动Worker服务
nohup bash scripts/dev/rundev-worker.sh > /tmp/rssant-worker.log 2>&1 &

# 启动Scheduler服务（关键！）
nohup bash scripts/dev/rundev-scheduler.sh > /tmp/rssant-scheduler.log 2>&1 &
```

### 3. 启动前端
```bash
cd frontend
npm run dev
```

## 🌐 访问地址

- **前端**: http://localhost:5173
- **API**: http://localhost:6788/api/v1/

## 📝 查看日志

```bash
tail -f /tmp/rssant-api.log        # API服务
tail -f /tmp/rssant-scheduler.log  # Scheduler服务
tail -f /tmp/rssant-worker.log     # Worker服务
tail -f /tmp/rssant-frontend.log   # 前端
```

## 🛑 停止服务

```bash
pkill -f "runserver"
pkill -f "run-scheduler"
pkill -f "vite"
docker stop rssant-postgres
```

## ⚠️ 常见问题

### 数据库连接失败
```bash
# 检查并启动数据库
docker ps | grep postgres || bash scripts/postgres_start.sh
```

### 任务一直pending
```bash
# 检查Scheduler是否运行
pgrep -f "run-scheduler" || nohup bash scripts/dev/rundev-scheduler.sh > /tmp/rssant-scheduler.log 2>&1 &
```

### 端口被占用
```bash
# 查找并停止占用端口的进程
lsof -i :6788 | grep LISTEN
kill -9 <PID>
```
