# 百家号参考链接解析问题排查与解决方案

> 更新时间：2025-11-20  
> 适用范围：`ai_entertainment.reference.preview`、`aigc.reference.preview`

## 1. 现象与根因

| 现象 | 根因 |
| --- | --- |
| 前端缩略卡片正常，点击详情弹窗显示“暂无法展示内容” | 百家号页面采用 JS 动态渲染，`requests.get()` 得到的 HTML 为空壳 |
| 接口返回 `"生成报告失败"` 或 `"Expecting value: line 1 column 1"` | Baijia号移动端接口偶发返回非 JSON（安全验证页 / HTML） |
| 返回 `"百度安全验证"` | 百家号触发风控校验，需要携带 Cookie 或使用浏览器渲染 |
| 某些链接 200 但正文依旧空 | 即使读取到 HTML，也只有安全验证页或极少量骨架，需要真实浏览器执行 JS |

## 2. 解决策略演进

1. **基础抓取**：`requests.get(url)` + Readability 解析。  
   - 适用于无需 JS 的站点。
2. **移动端接口兜底**：请求 `https://mbd.baidu.com/newspage/data/landingsuper?context=…`  
   - 通过 `context={"id":"文章ID"}` 获取 JSON，抽取 `data.content`。  
   - 解决了大多数 “动态渲染” 问题。
3. **编码与安全识别**：  
   - 统一 `quote` 编码 `context`，避免参数解析失败。  
   - 检测 `"百度安全验证" / verify.baidu.com` 关键字，遇到安全页立即切换备用方案。
4. **Cookie 预热**：百家号域名请求前，先访问 `https://www.baidu.com`，自动带上 `BAIDUID` 等 Cookie，降低触发安全验证概率。
5. **Playwright 兜底**（当前最终方案）：  
   - 当 PC 页 + 移动端接口都失败，启动 Headless Chromium 渲染完整 DOM，再交给 Readability/Markdown。  
   - 只在百家号且前两种策略失效时触发，控制性能成本。

## 3. 现行判定流程

1. `requests` 抓取 PC 页  
   - 失败 → 若非百家号，直接抛错  
   - 成功且非安全页 → Readability 解析
2. **（百家号专用）** 若 HTML 为空或命中安全页  
   - 用文章 ID 调 `mbd.baidu.com` 移动端接口  
   - 接口无结果 → 进入步骤 3
3. **Playwright** 渲染  
   - 抓取后再走 Readability/Markdown  
   - 若 Playwright 未安装，则记录日志并保留已有结果

## 4. 依赖与部署提示

- 需在后端环境安装 Playwright 及 Chromium：
  ```bash
  pip install playwright
  playwright install chromium
  ```
- Playwright 为可选依赖，未安装时兜底策略会自动降级（仅日志提醒）。

## 5. 常见错误对照

| 报错信息 | 意义与排查 |
| --- | --- |
| `"百度安全验证"` / 页面标题为“百度安全验证” | 已触发安全页，检查 Cookie/流量或等待 Playwright 兜底 |
| `"百家号移动端接口未返回正文内容"` | `mbd` 接口返回为空，自动进入 Playwright；如仍失败，多半需要代理或更长超时 |
| `"Expecting value: line 1 column 1"` | 已在移动端解析时捕获，这种报错不会再返回给前端，转为上面的友好提示 |

## 6. 未来优化方向

- 为 Playwright 补充代理池/多实例，提升通过率。
- 将百家号解析抽象成独立服务，供多模块复用。
- 缓存已解析过的链接，减少重复抓取。
