# 项目梳理文档

## 概览

- 项目名称：RSSAnt（蚁阅）扩展版，集成 AI 报告与飞书机器人能力
- 项目目标：提供稳定的 RSS 订阅、全文阅读、AI 行业报告生成与发布、飞书机器人推送等能力，支持容器化部署与本地开发
- 核心组成：后端（Django + DRF）、前端（React + Vite + Tailwind）、任务调度与 Worker、Cloudflare Worker 代理、容器与部署脚本

## 技术栈

- 后端：Django 2.x、Django REST framework、AllAuth、Whitenoise、PostgreSQL、supervisor
- 前端：React 18、TypeScript、Vite、Tailwind CSS、Radix UI/shadcn、Zustand、React Router、Axios
- 任务与调度：自研 `rssant_scheduler`，多角色运行（api/worker/scheduler）
- 构建与部署：Docker、多阶段镜像、`supervisord` 作业管理、脚本化部署
- 代理与辅助：Cloudflare Worker RSS 代理

## 目录结构

- `rssant/`：Django 项目入口与配置（URL、WSGI、中间件、模板等）
- `rssant_api/`：后端业务应用（Feed/Story/GitHub/HackerNews/ArXiv/AI 娱乐/AIGC/飞书机器人/发布等 API 视图与服务）
- `rssant_scheduler/`：调度服务与调度 API
- `frontend/`：前端工程（React + Vite + Tailwind）与页面/组件/状态管理
- `box/`：容器打包相关（Dockerfile、supervisord、启动脚本、Nginx、Postgres 初始化等）
- `deploy/`：生产部署脚本（API/Worker/AsyncAPI 构建与部署）
- `cloudflare_worker/rssant/`：Cloudflare Worker RSS 代理及文档
- `feishu/`：飞书相关脚本与集成示例
- `etc/`：OPML 分类源及系统配置（DNS、APT 源等）
- `scripts/`：开发与运维脚本（多服务启动、导入工具等）
- `tests/`：测试数据与配置（`pytest.ini`）

## 后端架构

- 项目设置：`rssant/settings/settings.py` 定义安装应用、中间件、静态文件、数据库配置、CSRF 信任来源、REST 框架设置等（rssant/settings/settings.py:1）
- URL 路由：`rssant/urls.py` 按角色生成 API 文档、Swagger、账户与业务接口并注册 `rssant_api`（rssant/urls.py:33）
- 角色与运行：通过环境配置决定角色（api / worker），WSGI 启动时初始化日志与 worker DNS 线程（rssant/wsgi.py:17）
- API 应用：`rssant_api/urls.py` 根据角色注册视图路由，如 `FeedView/StoryView/HackerNewsView/GitHubView/ArXivView/AIEntertainmentView/AIGCView/FeishuBotView/UserView/PublishView`（rssant_api/urls.py:23）
- 典型接口：
  - 订阅源管理：`feed.query`、`feed.get`、`feed.set_*`、`feed.delete` 等（rssant_api/views/feed.py:190）
  - 文章管理：`story.query`、`story.get`、收藏与已读设置等
  - 行业数据：`HackerNews`、`GitHub`、`ArXiv`、趋势分析与报告生成
  - AI 娱乐与 AIGC：搜索整合、报告生成服务（rssant_api/services/*）
  - 飞书机器人：配置创建/更新/删除/列表/获取等接口（rssant_api/views/feishu_bot.py:35）
  - 发布能力：匿名发布入口与鉴权（rssant_api/views/publish.py:1）

## 前端架构

- 技术栈：React 18 + TypeScript + Vite + Tailwind + Radix UI/shadcn + Zustand（frontend/package.json:12）
- 主要页面：订阅、详情、收藏、最近、设置、AI 娱乐、AIGC、飞书机器人配置等（frontend/src/pages/*）
- 组件与状态：`components/ui/*` 基础组件；`store/useAuthStore.ts`、`useThemeStore.ts` 管理认证与主题；`useServiceClientStore.ts` 管理服务客户端
- 构建脚本：`npm run dev/build/preview/lint`；API 基础地址通过 `VITE_API_BASE_URL` 配置（frontend/README.md:104）

## 任务调度与 Worker

- 调度服务：`rssant_scheduler/main.py` 以 `aiohttp` 运行，管理工作线程与对外 API（rssant_scheduler/main.py:1）
- 多角色运行：通过环境变量 `RSSANT_ROLE` 指定 `api`、`worker`、`scheduler`，分别监听 `6788/6793/6790`
- 开发脚本：`scripts/dev/rundev-api.sh`、`rundev-worker.sh`、`rundev-scheduler.sh`；`scripts/dev/start-all-services.sh` 支持统一启动与日志查看（scripts/dev/start-all-services.sh:201）

## 构建与部署

- 容器镜像：`box/Dockerfile` 多阶段构建前端与后端，安装 Nginx/Postgres 并以 `supervisord` 管理进程（box/Dockerfile:1）
- 进程管理：`box/supervisord.conf` 定义 `nginx/postgres/initdb/api/asyncapi/scheduler` 等程序（box/supervisord.conf:18）
- 运行脚本：`box/run.sh` 本地启动容器；`scripts/start_rssant.sh` 辅助启动与日志提示（scripts/start_rssant.sh:1）
- 部署脚本：`deploy/rssant_server/*.sh` 提供 API/Worker 构建与部署到平台的脚本（deploy/rssant_server/build.sh:1）

## RSS 代理（可选）

- Cloudflare Worker：`cloudflare_worker/rssant` 提供 RSS 代理，配置 `TOKEN` 与路由（cloudflare_worker/rssant/README.md:13）
- 环境变量示例（后端）：
  - `RSSANT_RSS_PROXY_ENABLE=true`
  - `RSSANT_RSS_PROXY_URL=https://<your-sub-domain>/rss-proxy`
  - `RSSANT_RSS_PROXY_TOKEN=<YOUR_TOKEN>`

## 配置与环境变量

- 示例文件：`box/rssant.env` 提供基础配置模板（box/rssant.env:1）
- 关键变量：
  - `RSSANT_DEBUG`：是否启用调试
  - `RSSANT_SECRET_KEY`：应用密钥（务必保密）
  - `RSSANT_ROOT_URL`：应用根地址
  - `RSSANT_CHECK_FEED_MINUTES`：订阅检查间隔
  - 邮件与 GitHub OAuth 相关配置
  - 数据库：`RSSANT_PG_DB/PG_HOST/PG_USER/PG_PASSWORD`
  - 角色与绑定：`RSSANT_ROLE`、`RSSANT_BIND_ADDRESS`
  - 可选：RSSHub 与 RSS 代理配置（不建议在仓库中存放任何真实密钥）

## 开发与运行

- 后端开发：
  - 运行 API：`python manage.py runserver 0.0.0.0:6788`
  - 运行 Worker：`RSSANT_ROLE=worker python manage.py runserver 0.0.0.0:6793`
  - 运行 Scheduler：`RSSANT_ROLE=scheduler python run-scheduler.py`
  - 推荐使用脚本：`scripts/dev/start-all-services.sh` 统一管理 screen 会话
- 前端开发：`cd frontend && npm install && npm run dev`，通过 `VITE_API_BASE_URL` 指向后端
- 测试：`pytest`（pytest.ini 设置了忽略目录与编码；DJANGO_SETTINGS_MODULE=rssant.settings）（pytest.ini:1）

## API 概览

- 文档访问（开发）：`/docs/v1/`、`/docs/v1/swagger/`（rssant/urls.py:46）
- 典型端点（前端 README 示例）：
  - `POST /api/v1/user/login/` 登录
  - `POST /api/v1/feed.query` 列表
  - `POST /api/v1/feed.get` 详情
  - `POST /api/v1/story.query` 列表
  - `POST /api/v1/story.get` 详情
  - `POST /api/v1/story.set_favorited` 收藏
  - `POST /api/v1/story.set_watched` 已读

## 安全与合规

- 严禁在仓库提交任何真实密钥或凭据（如 `RSSANT_SECRET_KEY`、飞书 App Secret、Webhook URL 等）
- 生产环境关闭 DEBUG，开启必要的 CSRF 与安全中间件，并限制来源
- 使用 `whitenoise` 提供静态资源，避免不必要的暴露
