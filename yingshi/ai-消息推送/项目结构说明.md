# 项目结构说明

## 模块划分

- 后端核心：`rssant/` 项目入口与配置；`rssant_api/` 业务应用与视图；`rssant_scheduler/` 调度服务。
- 前端：`frontend/` React + Vite 应用。
- 容器与部署：`box/`（Dockerfile、supervisord、Nginx、Postgres 初始化）、`deploy/`（构建与部署脚本）。
- 代理与集成：`cloudflare_worker/rssant/` RSS 代理；`feishu/` 飞书脚本。
- 配置与数据：`etc/` OPML 分类源与系统配置；`scripts/` 开发与运维脚本；`tests/` 测试数据与配置。

## 代码组织结构与分层

- 表现层：DRF 视图（`rssant_api/views/*`），前端页面（`frontend/src/pages/*`）。
- 领域层：Django ORM 模型与聚合（`rssant_api/models/*`）。
- 服务层：搜索、报告生成、RSSHub 客户端等（`rssant_api/services/*`）。
- 基础设施层：配置、日志、调度、容器启动与进程管理。

## 关键包说明

- `rssant/settings/settings.py`：Django 设置与中间件、数据库配置、REST 框架（rssant/settings/settings.py:1）。
- `rssant/urls.py`：路由注册与文档入口（rssant/urls.py:33）。
- `rssant_api/urls.py`：按角色注册各业务视图（rssant_api/urls.py:23）。
- `rssant_scheduler/main.py`：调度服务入口（rssant_scheduler/main.py:1）。
- `box/Dockerfile` 与 `box/supervisord.conf`：容器构建与进程编排（box/Dockerfile:1）。

## 分层架构说明

- 视图层调用聚合与服务层完成业务，领域模型负责数据一致性与约束，服务层封装跨模块逻辑（例如全文抓取、AI 报告生成、飞书推送）。
