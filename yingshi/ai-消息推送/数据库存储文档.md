# 数据库存储总结

## 概述

系统已完成从文件系统到数据库的迁移，所有分析报告、AI分析结果和相关文件都存储在数据库中，不再生成本地临时文件。

## 已完成的修改

### 1. ArXiv 报告系统
- ✅ HTML 生成器：已修改为不保存文件，返回 `None`
- ✅ 报告生成器：所有报告都保存到数据库，`report_file_path=None`
- ✅ 报告下载：从数据库读取内容，支持 Markdown、HTML、JSON 格式
- ✅ JSON 数据：存储在 `report_json_data` 字段中

### 2. Hacker News 报告系统
- ✅ 客户端导出：已修改为直接保存到数据库
- ✅ 报告生成器：从数据库读取原始数据，生成报告后保存到数据库
- ✅ 报告下载：从数据库读取报告内容
- ✅ 报告列表：从数据库查询，不依赖文件路径

### 3. GitHub 报告系统
- ✅ 报告生成器：已修改为从 `GitHubProgress` 对象生成报告（不依赖文件）
- ✅ 客户端导出：已废弃文件导出功能，返回 `None`
- ✅ 报告下载：从数据库读取报告内容
- ✅ 进度记录：`report_file_path` 设置为 `None`

### 4. AI 娱乐报告系统
- ✅ 报告生成器：`file_path=None`，所有内容保存到数据库
- ✅ 报告下载：从数据库读取报告内容
- ✅ 合并报告：独立报告和合并报告都存储在数据库中

### 5. AIGC 报告系统
- ✅ 报告生成器：`file_path=None`，所有内容保存到数据库
- ✅ 报告下载：从数据库读取报告内容
- ✅ 参考链接：存储在 `aigc_refs` JSON 字段中

## 数据库字段说明

### 报告文件路径字段
所有模型的 `report_file_path` 字段保留用于向后兼容，但值始终为 `None`：
- `ArXivReport.report_file_path = None`
- `HackerNewsReport.report_file_path = None`
- `GitHubReport.report_file_path = None`
- `AIEntertainmentReport.report_file_path = None`
- `AIGCReport.report_file_path = None`

### 内容存储字段
所有报告内容都存储在以下字段中：
- `report_content` (TextField): Markdown 格式的报告内容
- `report_json_data` (JSONField): JSON 格式的原始数据（ArXiv 报告）
- `ai_entertainment_report_content` (TextField): AI 影视独立报告
- `aigc_report_content` (TextField): AIGC 独立报告

## 导入功能说明

以下功能保留文件读取能力，用于将已有文件导入数据库（一次性迁移）：
- `hacker_news.report.import_from_file`: 导入单个报告文件
- `scripts/import_hacker_news_reports_to_db.py`: 批量导入脚本
- `scripts/import_arxiv_reports_to_db.py`: ArXiv 报告导入脚本
- `rssant_api/management/commands/import_hacker_news_reports.py`: Django 管理命令

这些功能仅用于迁移，不会在正常业务流程中产生文件。

## 模板文件说明

以下文件写入是必要的（模板文件，不是报告文件）：
- `rssant_api/services/arxiv_html_generator.py`: 创建 HTML 模板文件（仅首次运行时）

## 下载功能说明

所有下载功能都从数据库读取内容：
- `hacker_news.report.download`: 从数据库读取并返回文件
- `arxiv.report.download`: 从数据库读取，支持多种格式
- `github.report.download`: 从数据库读取
- `ai_entertainment.report.download`: 从数据库读取
- `aigc.report.download`: 从数据库读取

## 验证清单

- ✅ 所有报告生成器都不写入文件
- ✅ 所有下载功能都从数据库读取
- ✅ 所有报告内容都存储在数据库中
- ✅ 没有代码尝试从 `report_file_path` 读取文件

## 注意事项

1. **向后兼容性**: `report_file_path` 字段保留在数据库中，但始终为 `None`
2. **导入功能**: 文件导入功能保留，但仅用于一次性迁移
3. **模板文件**: HTML 模板文件的创建是必要的，这些不是报告文件
4. **数据库存储**: 所有报告内容存储在数据库中，可以通过 API 查询和下载

## 迁移建议

如果您之前有本地报告文件，可以使用以下命令导入到数据库：
- `python manage.py import_hacker_news_reports --hacker-news-dir <目录路径>`
- `python scripts/import_arxiv_reports_to_db.py <目录路径>`

导入完成后，可以安全删除本地文件（建议先验证数据库中的数据）。
