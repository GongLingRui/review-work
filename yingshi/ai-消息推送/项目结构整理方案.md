## 执行步骤

### 阶段一：创建新目录结构
1. 创建 `tests/integration/`
2. 创建 `tests/fixtures/` 和 `tests/fixtures/screenshots/`
3. 创建 `scripts/docker/`
4. 创建 `requirements/`（如果需要）

### 阶段二：迁移测试文件
1. 移动根目录测试文件到 `tests/integration/`
2. 移动测试数据到 `tests/fixtures/`
3. 移动 feishu 测试文件
4. 移动 scripts 测试文件

### 阶段三：迁移脚本文件
1. 移动启动脚本到 `scripts/`
2. 移动工具脚本到 `scripts/tools/`
3. 移动 Docker 脚本到 `scripts/docker/`
4. 删除根目录的重复脚本

### 阶段四：整理数据文件
1. 移动 `hacker_news/` 到 `data/hacker_news/`
2. 移动 `logs/` 到 `data/logs/`（如果不在 .gitignore）
3. 确认 `.gitignore` 包含日志文件

### 阶段五：整理文档
1. 创建 `docs/technical/`
2. 迁移 `cursor/docs/` 技术文档到 `docs/technical/`
3. 创建文档索引

### 阶段六：更新引用
1. 更新所有脚本中的路径引用
2. 更新 README.md 中的路径说明
3. 更新 pytest.ini 配置（如果需要）
4. 检查并更新所有导入路径

### 阶段七：验证
1. 运行测试确保路径正确
2. 运行启动脚本确保正常工作
3. 检查 Git 状态，确认没有遗漏文件

## 注意事项

1. **备份**：在执行迁移前，建议先提交当前代码或创建备份分支
2. **逐步迁移**：建议分阶段执行，每完成一个阶段就测试一次
3. **更新文档**：迁移后及时更新 README.md 和相关文档
4. **Git 历史**：使用 `git mv` 命令移动文件以保留 Git 历史
5. **依赖检查**：迁移后检查是否有硬编码路径需要更新

## 可选优化

1. **创建 scripts/bin/ 目录**：将常用脚本添加到 PATH
2. **统一脚本命名**：使用统一的前缀或命名规范
3. **添加 Makefile**：提供常用命令的快捷方式
4. **文档自动化**：考虑使用 Sphinx 或 MkDocs 生成文档网站
