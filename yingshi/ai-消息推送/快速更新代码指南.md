# 快速更新代码指南

## 问题说明

Docker 容器中的代码是在构建镜像时打包进去的静态文件，修改源代码后需要重新构建镜像才能看到更新。

## 解决方案

### 方案1: 使用开发环境（推荐 - 最快）⚡

**前端开发服务器支持热更新，代码修改后自动刷新！**

#### 启动前端开发服务器

```bash
cd /root/gongfan/rrs-aigc/frontend
bash dev-update.sh
```

或者手动启动：

```bash
cd /root/gongfan/rrs-aigc/frontend
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
nvm use 18
npm run dev
```

#### 访问地址

- **开发环境**: http://192.168.191.85:5173
- **生产环境**: http://192.168.191.85:6789 (Docker容器)

#### 优点

- ✅ **热更新**: 代码修改后自动刷新，立即看到效果
- ✅ **快速**: 无需重建镜像，几秒内生效
- ✅ **开发友好**: 完整的错误提示和调试信息

#### 注意事项

- 开发服务器运行在 5173 端口
- 修改前端代码会自动热更新
- 修改后端代码仍需要重启 Docker 容器

---

### 方案2: 重新构建 Docker 镜像（生产环境）🏗️

如果需要更新 Docker 容器中的代码，需要重新构建镜像。

#### 快速重建脚本

```bash
cd /root/gongfan/rrs-aigc/box
bash build-and-restart.sh
```

这个脚本会：
1. 停止当前容器
2. 重新构建镜像（包含所有最新代码）
3. 启动新容器

#### 手动重建步骤

```bash
# 1. 停止并删除容器
docker stop rssant
docker rm rssant

# 2. 重新构建镜像
cd /root/gongfan/rrs-aigc/box
docker build -t rssant:custom -f Dockerfile ..

# 3. 启动容器
bash run.sh
```

#### 优点

- ✅ **生产环境**: 完整的生产环境配置
- ✅ **稳定性**: 所有依赖都已打包

#### 缺点

- ⏱️ **耗时**: 构建镜像需要几分钟时间
- 🔄 **需要重启**: 每次修改都要重建

1. **使用开发服务器**（5173端口）

2. **重启 Docker 容器**


1. **前端代码更新**

   ```bash
   docker restart rssant

   cd /root/gongfan/rrs-aigc/box
   bash build-and-restart.sh
   ```

## 常见问题

### Q: 修改代码后看不到效果？

**A**: 
- 如果使用开发环境（5173端口），检查浏览器是否自动刷新
- 如果使用生产环境（6789端口），需要重新构建镜像

### Q: 开发环境和生产环境有什么区别？

**A**:
- **开发环境（5173）**: 
  - 热更新支持
  - 完整的错误提示
  - 开发工具支持
  - 适合日常开发

- **生产环境（6789）**:
  - 代码已优化打包
  - 性能更好
  - 适合最终部署

### Q: 如何同时使用两个环境？

**A**: 
- 开发环境用于快速开发和测试
- 生产环境用于演示和最终部署
- 两个环境可以同时运行，访问不同端口即可
