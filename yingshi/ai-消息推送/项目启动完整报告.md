## 项目概述

**项目名称**: RSS订阅系统 (RSSAnt)

**项目描述**: 一个基于Django和React的RSS订阅管理系统，支持文章爬取、AI分析、翻译等功能。

**技术栈**:
- **后端**: Django 3.x, Python 3.8+
- **前端**: React 18, TypeScript, Vite
- **数据库**: PostgreSQL 11
- **任务调度**: 自定义Scheduler服务
- **AI服务**: 智谱AI (GLM-Z1-Flash)

---

## 系统架构

### 服务组件

项目采用微服务架构，包含以下核心服务：

1. **API服务** (端口 6788)
   - 处理前端HTTP请求
   - 提供RESTful API接口
   - 用户认证和授权

2. **Worker服务** (端口 6793)
   - 执行RSS订阅源抓取任务
   - 处理文章内容提取
   - 执行后台异步任务

3. **Scheduler服务** (端口 6790)
   - **关键服务**，负责任务调度
   - 从数据库队列获取任务
   - 分发任务给Worker服务
   - 定时执行文章处理任务（初评、分析、翻译）

4. **前端服务** (端口 5173)
   - React单页应用
   - Vite开发服务器
   - 提供用户界面

5. **PostgreSQL数据库** (端口 5432)
   - 存储所有业务数据
   - 任务队列存储
   - 文章内容存储

### 服务依赖关系

```
前端 (5173) 
    ↓ HTTP请求
API服务 (6788)
    ↓ 创建任务
数据库 (5432)
    ↑ 读取任务
Scheduler (6790)
    ↓ 分发任务
Worker服务 (6793)
    ↓ 执行任务
外部RSS源
```

## 数据库配置

### 方法1: 使用Docker启动PostgreSQL (推荐)

#### 1. 启动Docker Desktop

确保Docker Desktop应用正在运行。

#### 2. 启动PostgreSQL容器

```bash
 cd /path/to/rss-subscribe
bash scripts/postgres_start.sh
```

这个脚本会：
- 创建Docker网络 `rssant`
- 创建数据卷 `rssant_postgres`
- 启动PostgreSQL 11.7容器
- 配置数据库：用户 `rssant`，密码 `rssant`，数据库 `rssant`
- 映射端口 5432

#### 3. 验证数据库运行

```bash
# 检查容器状态
docker ps | grep postgres

# 检查端口监听
lsof -i :5432

# 测试连接
pg_isready -h localhost -p 5432
```

### 方法2: 使用Homebrew安装PostgreSQL

```bash
# 安装PostgreSQL
brew install postgresql@11

# 启动服务
brew services start postgresql@11

# 创建数据库和用户
psql postgres
CREATE USER rssant WITH PASSWORD 'rssant';
CREATE DATABASE rssant OWNER rssant;
\q
```

### 运行数据库迁移

```bash
# 激活conda环境
conda activate rssant

# 运行迁移
python manage.py migrate
```

这会创建所有必要的数据库表，包括：
- 用户和认证表
- Feed和Story表
- AI分析相关表
- 任务队列表

## 环境变量配置

### 创建.env文件

在项目根目录创建 `.env` 文件：

```bash
# 基础配置
RSSANT_DEBUG=0
RSSANT_SECRET_KEY=your-secret-key-here  # 请随机生成
RSSANT_ROOT_URL=http://localhost:6789

# 数据库配置
RSSANT_PG_HOST=localhost
RSSANT_PG_PORT=5432
RSSANT_PG_DB=rssant
RSSANT_PG_USER=rssant
RSSANT_PG_PASSWORD=rssant

# 服务配置
RSSANT_HARBOR_URL=http://localhost:6788
RSSANT_WORKER_URL=http://localhost:6793
RSSANT_CHECK_FEED_MINUTES=30

# AI服务配置（可选，用于文章AI分析功能）
RSSANT_AI_API_KEY=your-zhipu-api-key
RSSANT_AI_API_BASE_URL=https://open.bigmodel.cn/api/paas/v4
RSSANT_AI_TRANSLATE_MODEL_CONFIG=glm-z1-flash,glm-z1-flash

# RSSHub配置（可选）
RSSANT_RSSHUB_BASE_URL=https://rsshub.app
RSSANT_RSSHUB_ENABLE=true
RSSANT_RSSHUB_TIMEOUT=30

# GitHub OAuth（可选）
RSSANT_GITHUB_CLIENT_ID=
RSSANT_GITHUB_SECRET=

# SMTP邮件配置（可选）
RSSANT_SMTP_ENABLE=false
RSSANT_SMTP_HOST=smtp.qq.com
RSSANT_SMTP_PORT=465
RSSANT_SMTP_USE_SSL=true
RSSANT_SMTP_USERNAME=
RSSANT_SMTP_PASSWORD=
```

### 重要配置说明

1. **SECRET_KEY**: 用于Django安全加密，必须随机生成
   ```bash
   python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
   ```

2. **AI_API_KEY**: 如果使用AI分析功能，需要配置智谱AI的API密钥

3. **数据库密码**: 确保与PostgreSQL配置一致

## 启动流程

### 快速启动（推荐）

使用提供的启动脚本：

```bash
 cd /path/to/rss-subscribe
bash start-project.sh
```

这个脚本会自动：
1. 检查并激活conda环境
2. 检查依赖
3. 启动PostgreSQL（如果未运行）
4. 启动所有后端服务
5. 启动前端服务

### 手动启动步骤

#### 步骤1: 启动数据库

```bash
# 如果使用Docker
bash scripts/postgres_start.sh

# 验证数据库运行
lsof -i :5432
```

#### 步骤2: 激活环境

```bash
source $(conda info --base)/etc/profile.d/conda.sh
conda activate rssant
```

#### 步骤3: 启动后端服务

**启动API服务**:
```bash
 cd /path/to/rss-subscribe
nohup bash scripts/dev/rundev-api.sh > /tmp/rssant-api.log 2>&1 &
```

**启动Worker服务**:
```bash
nohup bash scripts/dev/rundev-worker.sh > /tmp/rssant-worker.log 2>&1 &
```

**启动Scheduler服务** (关键):
```bash
nohup bash scripts/dev/rundev-scheduler.sh > /tmp/rssant-scheduler.log 2>&1 &
```

#### 步骤4: 启动前端服务

```bash
cd frontend
npm run dev
```

前端服务会在 `http://localhost:5173` 启动。

### 使用Screen管理服务（生产环境推荐）

```bash
# 安装screen（如果未安装）
# macOS: brew install screen
# Linux: sudo apt-get install screen

# 使用提供的脚本
bash scripts/dev/start-all-services.sh start
```

### 验证服务启动

```bash
# 检查所有服务状态
bash check-services.sh

# 或手动检查
lsof -i :6788  # API服务
lsof -i :6793  # Worker服务
```

## 常见问题

### Q1: 数据库连接失败

**症状**: 
```
django.db.utils.OperationalError: connection to server at "localhost" (::1), port 5432 failed
```

**解决方案**:
1. 检查PostgreSQL是否运行: `docker ps | grep postgres`
2. 检查端口是否监听: `lsof -i :5432`
3. 如果未运行，启动数据库: `bash scripts/postgres_start.sh`
4. 检查数据库配置: 确认 `.env` 文件中的数据库配置正确

### Q2: 任务一直处于pending状态

**症状**: 导入订阅后，任务一直显示"处理中"

**原因**: Scheduler服务未运行

**解决方案**:
```bash
# 检查Scheduler是否运行
pgrep -f "run-scheduler"

# 如果未运行，启动它
nohup bash scripts/dev/rundev-scheduler.sh > /tmp/rssant-scheduler.log 2>&1 &
```

### Q3: 前端无法连接后端

**症状**: 前端显示"无法连接到后端服务"

**解决方案**:
1. 检查API服务是否运行: `lsof -i :6788`
2. 检查后端日志: `tail -f /tmp/rssant-api.log`
3. 确认前端配置的API地址正确（默认: `/api/v1`）

### Q4: 端口被占用

**症状**: 
```
Error: listen EADDRINUSE: address already in use :::6788
```

**解决方案**:
```bash
# 查找占用端口的进程
lsof -i :6788

# 停止占用端口的进程
kill -9 <PID>
```

### Q5: Conda环境激活失败

**症状**: 
```
CommandNotFoundError: Your shell has not been properly configured to use 'conda activate'
```

**解决方案**:
```bash
# 初始化conda
source $(conda info --base)/etc/profile.d/conda.sh
conda activate rssant
```

### Q6: 数据库迁移失败

**症状**: 
```
django.db.utils.ProgrammingError: column "xxx" does not exist
```

**解决方案**:
```bash
# 运行数据库迁移
conda activate rssant
python manage.py migrate
```

### Q7: AI服务未配置

**症状**: 
```
AI服务初始化: API Key未配置，将使用模拟数据
```

**说明**: 这是正常的，如果不使用AI功能，可以忽略。如需使用AI功能，在 `.env` 文件中配置 `RSSANT_AI_API_KEY`。

## 故障排查

### 系统检查清单

在启动项目前，请确认：

- [ ] Python 3.8+ 已安装
- [ ] Conda环境已创建并激活
- [ ] Python依赖已安装 (`pip install -r requirements.txt`)
- [ ] Node.js 18+ 已安装
- [ ] 前端依赖已安装 (`cd frontend && npm install`)
- [ ] Docker Desktop正在运行
- [ ] PostgreSQL数据库已启动
- [ ] 数据库迁移已执行 (`python manage.py migrate`)
- [ ] `.env` 文件已配置
- [ ] 所有端口未被占用 (6788, 6793, 5173, 5432)

### 日志分析

#### API服务错误

查看 `/tmp/rssant-api.log`:
- `500 Internal Server Error`: 检查数据库连接和代码错误
- `404 Not Found`: 检查路由配置
- `401 Unauthorized`: 检查认证token

#### Scheduler服务错误

查看 `/tmp/rssant-scheduler.log`:
- 任务分发失败: 检查Worker服务是否运行
- 数据库连接错误: 检查数据库配置

#### Worker服务错误

查看 `/tmp/rssant-worker.log`:
- RSS抓取失败: 检查网络连接
- 内容解析错误: 检查feedlib配置

### 性能优化建议

1. **数据库优化**
   - 定期清理旧文章: 配置 `RSSANT_FEED_STORY_RETENTION`
   - 使用数据库索引: 已自动创建

2. **服务并发**
   - 调整Scheduler工作线程: `RSSANT_SCHEDULER_NUM_WORKER`
   - 限制同时处理的文章数

3. **资源监控**
   - 监控内存使用: `top` 或 `htop`
   - 监控数据库连接: `psql -c "SELECT count(*) FROM pg_stat_activity;"`

## 总结

本项目采用微服务架构，需要同时运行多个服务组件。关键要点：

1. **Scheduler服务是关键**: 必须运行才能处理任务
2. **数据库必须先启动**: 所有服务都依赖PostgreSQL
3. **环境配置很重要**: 确保 `.env` 文件配置正确
4. **日志是排查问题的关键**: 遇到问题先查看日志

建议使用提供的启动脚本 `start-project.sh` 或 `scripts/dev/start-all-services.sh` 来管理服务，这样可以确保所有服务正确启动和配置。
